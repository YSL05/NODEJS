Git版本使用说明

Git版本控制优势

  1. 操作简单
  2. 易于对比
  3. 易于回溯
  4. 协作方便

git借助快照的思想管理项目中的不同版本，所有的操作都是在本地执行的，开发在本地进行，服务器用于存储项目文件

Git操作步骤

  1. 第一步 配置全局设置 配置用户名 git config --global user.name "你的名字" 配置用户邮箱 git config --global user.email "你的邮箱"
  2. 第二步 查看配置 git config --list --global \ git config user.name
  3. 第三步 获取常用命令帮助信息 git help config
  4. 第四步 与git仓库进行交互
    1. 第一步 初始化 git init :将当前目录转化为git仓库
    2. 第二步 git status / git add 文件名 #三个功能 跟踪新文件 将修改文件放到暂存区 / git add . / git commit -m "修改描述"
    3. 第三步 git reset HEAD 要移除文件名字 # 移除暂存区的文件 / git reset HEAD . / git commit -a -m "" 跳过暂存区直接提交
    4. 第四步 git rm -f 要移除文件的名字 #移除工作区和仓库中文件 / git rm --cached 文件名字 #从仓库中移除文件
    5. 第五步 回顾提交历史 git log / git log -2 / git log -2 --pretty=oneline / git log -2 --pretty=format:"%h | %an | %ar | %s"
    6. 第六步 git reset --hard 提交命令名 / git relog --pretty=oneline #查看命令操作历史
  5. 第五步 使用https与github进行交互
    1. 第一步 git remote add origin 仓库https的地址 / git push -u origin main
    2. 第二步 #将本地文件修改推到远程仓库中 先进行本地仓库的commit 在进行push git push
  6. 第六步 使用SSH与github交互
    1. 第一步 生成SSH密匙 ssh-keygen -t rsa -b 4096 -C "你的github邮箱名字" / 连续敲击三次回车即可在C：\User\用户名文件夹\.ssh
    2. 第二步 在路径中打开id_rsa.pub文件 在github中 setting/SSH and GPG keys/New SSH key
    3. 第三步 检测是否配置成功 ssh -T git@github.com /
    4. 第四步 将本地仓库通过SSH推到远程仓库
  7. 第七步 将远程仓库同步到本地（包含分支）
    1. git clone 远程仓库地址
    2. 分支的含义 master用于保存和记录整个项目已完成的功能 分支用于开发测试新功能
    3. git branch #查看所有分支 / git branch 分支名字 #创建新分支 / git checkout 分支名字 #切换到分支上 / git checkout -b 分支名字 #创建和切换
    4. 分支合并 / git merge 分支名字 / git branch -d 分支名字 #删除分支 /
    5. 分支冲突(两个分支对同一个文件进行修改并进行提交) 手动解决冲突 再进行上传（add commit）
    6. 推送分支 git push -u 本地分支名字：远程仓库分支名字 下载远程仓库分支 git checkout 远程仓库分支名字 / git checkout -b 本地仓库名字 origin/远程仓库名字
    7. 拉取远程仓库分支 git pull
