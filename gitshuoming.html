Git版本使用说明

Git版本控制优势

、、
  1. 操作简单
  2. 易于对比
  3. 易于回溯
  4. 协作方便

git借助快照的思想管理项目中的不同版本，所有的操作都是在本地执行的，开发在本地进行，服务器用于存储项目文件

Git操作步骤

  1. 第一步 配置全局设置 配置用户名 git config --global user.name "你的名字" 配置用户邮箱 git config --global user.email "你的邮箱"
  2. 第二步 查看配置 git config --list --global \ git config user.name
  3. 第三步 获取常用命令帮助信息 git help config
  4. 第四步 与git仓库进行交互
    1. 第一步 初始化 git init :将当前目录转化为git仓库
    2. 第二步 git status / git add 文件名 #三个功能 跟踪新文件 将修改文件放到暂存区 / git add . / git commit -m "修改描述"
    3. 第三步 git reset HEAD 要移除文件名字 # 移除暂存区的文件 / git reset HEAD . / git commit -a -m "" 跳过暂存区直接提交
    4. 第四步 git rm -f 要移除文件的名字 #移除工作区和仓库中文件 / git rm --cached 文件名字 #从仓库中移除文件
    5. 第五步 回顾提交历史 git log / git log -2 / git log -2 --pretty=oneline / git log -2 --pretty=format:"%h | %an | %ar | %s"
    6. 第六步 git reset --hard 提交命令名 / git relog --pretty=oneline #查看命令操作历史
  5. 第五步 与github进行交互
