Node.js的学习路径

Javascript基础语法 + 内置API接口fs、http + 第三方框架

终端常用操作: 上箭头 历史命令 TAB 补全路径 ESC 清除命令 cls 清空终端

fs文件系统模块

  1. fs.readFile(path[,option],callback) 读取文件内容
  2. fs.writefile(file,data[,options],callback) 写入文件内容 data 为写入数据 []为可选参数,会覆盖掉原本的内容

path路径模块

  1. path.join(path,path,path,) 将多个路径模块拼接成一个完整的路径
  2. path.basename(path[,'.扩展名']) 获取路径的最后一个部分，也可以去除扩展名
  3. path.extname() 获取路径中的扩展名

HTTP模块: 创建Web服务器

  1. IP地址
  2. 域名与域名服务器
  3. 端口号 对应web服务；一个端口号对应一个web服务；在实际应用中，URL中端口80可以被省略；
  4. req 请求对象 包含与客户端有关的数据和属性：url地址 method 请求类型
  5. res 响应对象

模块化基本概念

  1. 模块来源不同：内置模块 自定义模块 第三方模块
  2. 模块作用域 内部定义变量不会被外部使用
  3. 通过module.exports向外共享属性和方法

npm与包

  1. npm install 包的名字
  2. const 包名 = require('包名')
  3. 2.24.0 第一位数字代表大版本 第二位数字代表功能版本 第三位数字代表BUG修复版本
  4. npm i 通过package.json文件一次性安装所有包
  5. npm uninstall 包名 删除包
  6. 项目上线后不使用 仅开发时使用，开发包 npm install -D 包名
  7. npm config get registry 查看下包的地址 npm config set registry=http://... 代表更换地址
  8. nrm 包下载地址管理 npm i nrm -g 安装 nrm ls 查看所有包地址 nrm use taobao 使用淘宝镜像源
  9. i5ting_toc包 将.md文件转换为.html文件 安装命令 npm install -g i5ting_toc 使用命令 i5ting_toc -f md文件路径 -o
  10. 包的规范结构 包单独目录存在 包中包含 name version main三个属性 包顶级目录在package.json里面

Express框架学习: 基于node.js构建的web框架 类似http 用于构架web服务器和api接口服务器

  1. 通过npm安装express npm i express
  2. express.static()托管静态资源 app.use(express.static('./clock')) / app.use('/clock',express.static('./clock'))
  3. nodemon 管理工具 不需要每次修改文件再重新运行文件
  4. app.use 注册中间件
  5. 中间件 function(req,res,next){ next() } 多个中间件采用相同的req和res，上游的req对象下游可以使用，多个中间件按照定义顺序处理，不使用app.use是局部中间件
  6. 路由 路由 映射关系 直接挂载到服务器上和路由模块导入
  7. 内置中间件 static快速托管静态资源 express.json 解析JSON格式的请求体数据 urlencoded 解析URL格式的请求体数据
  8. 第三方中间件
  9. Express接口 基于CORS解决跨域问题 cors为Express的第三方包 在路由模块之前使用cors模块解决跨域问题 安装 导入 使用 app.use(cors()) 注意事项 在服务器端配置 兼容问题 XMLHttpRequestLevel 2才能使用
